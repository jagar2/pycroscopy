_get_endian_str(endian)[source]¶

DM4 header encodes little endian as byte value 1 in the header :return: ‘big’ or ‘little’ for use with python’s int.frombytes function

_get_struct_endian_str(endian)[source]¶

DM4 header encodes little endian as byte value 1 in the header. However when that convention is followed the wrong values are read. So this implementation is reversed. :return: ‘>’ or ‘<’ for use with python’s struct.unpack function
